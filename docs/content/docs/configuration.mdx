----
This page documents all configuration options for `syncgen`.

## Top-Level Keys
### `primary` (object, required)
- `host` (string, required): IP or hostname of the primary server
- `port` (integer, default: 5432): PostgreSQL port
- `data_directory` (string, required): Path to PostgreSQL data directory
- `db_name` (string, required): Name of the main database
- `db_user` (string, required): Admin user for the database
- `db_password` (string, required): Password for `db_user`
- `replication_user` (string, required): Replication user name
- `replication_password` (string, required): Replication user password

### `replicas` (array of objects, required)
Each replica must have a unique `replication_slot` name.
- `host` (string, required): IP or hostname of the replica
- `port` (integer, default: 5432): PostgreSQL port
- `replication_slot` (string, required): Unique slot name for this replica
- `sync_mode` (string, required): `sync` or `async`
- `db_user` (string, required): Admin user for the replica
- `db_password` (string, required): Password for `db_user`

### `options` (object, required)
- `promote_on_failure` (bool, default: true): Auto-promote replica on primary failure
- `wal_level` (string, default: `replica`): WAL verbosity (`replica`, `logical`, etc.)
- `max_wal_senders` (integer, default: 3): Max WAL sender processes
- `wal_keep_size` (string, default: `1GB`): WAL retention size (e.g., `1GB`, `2GB`)
- `hot_standby` (bool, default: true): Allow read queries on replicas
- `synchronous_commit` (string, default: `on`): Synchronous commit mode (`on`, `off`, `remote_write`, etc.)

### `monitoring` (object, optional)
- `datadog` (object, optional):
	- `enabled` (bool, default: false): Enable Datadog integration
	- `api_key` (string, required if enabled): Datadog API key
	- `site` (string, default: `datadoghq.com`): Datadog site
	- `datadog_user_password` (string, required if enabled): Password for Datadog agent user

---
## Example
```yaml
primary:
	host: 10.0.0.1
	port: 5432
	data_directory: /var/lib/postgresql/data
	db_name: postgres
	db_user: postgres
	db_password: change_this_admin_password
	replication_user: replicator
	replication_password: secure_password_here

replicas:
	- host: 10.0.0.2
		port: 5432
		replication_slot: replica_slot_1
		sync_mode: async
		db_user: postgres
		db_password: change_this_admin_password
	- host: 10.0.0.3
		port: 5432
		replication_slot: replica_slot_2
		sync_mode: async
		db_user: postgres
		db_password: change_this_admin_password

options:
	promote_on_failure: true
	wal_level: replica
	max_wal_senders: 3
	wal_keep_size: 1GB
	hot_standby: true
	synchronous_commit: on

monitoring:
	datadog:
		enabled: true
		api_key: your_datadog_api_key_here
		site: datadoghq.com
		datadog_user_password: change_this_monitoring_password
```
