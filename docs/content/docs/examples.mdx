
# Examples

Here are five valuable real-world configuration examples for `syncgen`.

---

## 1. Minimal 2-Replica Cluster

```yaml
primary:
	host: 10.0.0.1
	port: 5432
	data_directory: /var/lib/postgresql/data
	db_name: postgres
	db_user: postgres
	db_password: password
	replication_user: replicator
	replication_password: replica_pw

replicas:
	- host: 10.0.0.2
		port: 5432
		replication_slot: replica1
		sync_mode: async
		db_user: postgres
		db_password: password
	- host: 10.0.0.3
		port: 5432
		replication_slot: replica2
		sync_mode: async
		db_user: postgres
		db_password: password

options:
	promote_on_failure: true
	wal_level: replica
	max_wal_senders: 3
	wal_keep_size: 1GB
	hot_standby: true
	synchronous_commit: on
```

---

## 2. Production Cluster with Synchronous and Asynchronous Replicas

```yaml
primary:
	host: db-primary.internal
	port: 5432
	data_directory: /var/lib/postgresql/data
	db_name: prod_db
	db_user: admin
	db_password: prod_pw
	replication_user: replicator
	replication_password: prod_replica_pw

replicas:
	- host: db-replica1.internal
		port: 5432
		replication_slot: prod_sync
		sync_mode: sync
		db_user: admin
		db_password: prod_pw
	- host: db-replica2.internal
		port: 5432
		replication_slot: prod_async
		sync_mode: async
		db_user: admin
		db_password: prod_pw

options:
	promote_on_failure: true
	wal_level: replica
	max_wal_senders: 5
	wal_keep_size: 4GB
	hot_standby: true
	synchronous_commit: remote_apply

monitoring:
	datadog:
		enabled: true
		api_key: prod_datadog_api_key
		site: datadoghq.com
		datadog_user_password: prod_monitor_pw
```

## 3. Development Cluster (Single Replica, Fast Settings)

```yaml
primary:
	host: localhost
	port: 5432
	data_directory: /var/lib/postgresql/data
	db_name: dev_db
	db_user: dev
	db_password: dev_pw
	replication_user: dev_repl
	replication_password: dev_repl_pw

replicas:
	- host: localhost
		port: 5433
		replication_slot: dev_replica
		sync_mode: async
		db_user: dev
		db_password: dev_pw

options:
	promote_on_failure: false
	wal_level: replica
	max_wal_senders: 2
	wal_keep_size: 512MB
	hot_standby: true
	synchronous_commit: off
```

## 5. High-Security Cluster (Unique Users, Strong Passwords, No Monitoring)

```yaml
primary:
	host: 192.168.1.10
	port: 5432
	data_directory: /secure/pg/data
	db_name: secure_db
	db_user: secure_admin
	db_password: "!@#SuperStrongPW2025"
	replication_user: secure_repl
	replication_password: "!@#ReplPW2025"

replicas:
	- host: 192.168.1.11
		port: 5432
		replication_slot: secure_replica1
		sync_mode: sync
		db_user: secure_replica
		db_password: "!@#ReplicaPW2025"
	- host: 192.168.1.12
		port: 5432
		replication_slot: secure_replica2
		sync_mode: async
		db_user: secure_replica
		db_password: "!@#ReplicaPW2025"

options:
	promote_on_failure: true
	wal_level: logical
	max_wal_senders: 6
	wal_keep_size: 2GB
	hot_standby: true
	synchronous_commit: on
```
